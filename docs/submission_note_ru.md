# Пояснительная записка

## Проект

**Веб-приложение для проведения интерактивных опросов и квизов**

## 1. Цель разработки

Цель проекта состояла в создании веб-приложения, которое позволяет организатору быстро собирать и проводить квизы, а участнику подключаться к ним, отвечать на вопросы и видеть результаты. Основной акцент был сделан на поддержку двух сценариев работы:

- обычное прохождение квиза в индивидуальном режиме;
- live-режим с синхронным показом вопросов для нескольких участников.

## 2. Функциональный результат

В готовой версии реализованы следующие возможности:

- регистрация и авторизация пользователей с разделением ролей `organizer` и `participant`;
- создание и редактирование квизов;
- настройка параметров квиза, лимитов, таймеров и правил показа ответов;
- загрузка изображений для вопросов;
- запуск live-сессий и подключение участников по коду комнаты;
- синхронное обновление состояния квиза через WebSocket;
- отображение лидерборда и сохранение истории прохождений;
- редактирование профиля и загрузка аватара;
- автоматизированная проверка качества через unit, integration и browser e2e тесты.

## 3. Обоснование выбора средств разработки

### Frontend

Для клиентской части использован **React**, поскольку он позволяет строить SPA-приложение из переиспользуемых компонентов и удобно разделять пользовательские сценарии для разных ролей.

Для маршрутизации использован **react-router-dom**, что позволило реализовать кабинеты организатора и участника, а также защищенные маршруты.

Для сборки выбран **Vite**, так как он обеспечивает быстрый локальный запуск, современную сборку frontend-части и удобный production preview.

### Backend

Серверная часть реализована на **Node.js + Express**. Этот стек подходит для построения REST API, работы с middleware и удобной интеграции с JavaScript-экосистемой.

Для поддержки live-режима использована библиотека **ws**, которая позволяет организовать двустороннюю связь между сервером и клиентом и синхронно обновлять состояние сессии.

### База данных

В качестве СУБД выбрана **PostgreSQL**. Она подходит для хранения структурированных данных о пользователях, квизах, вопросах, попытках и live-сессиях, поддерживает надежные связи между сущностями и удобна для аналитических запросов.

### Тестирование и контроль качества

Для unit- и component-тестов использованы **Jest** и **Testing Library**.

Для интеграционных проверок backend использованы smoke-тесты реальных API-маршрутов.

Для browser smoke-тестов использован **Playwright**, что позволило проверить ключевые сценарии в настоящем браузере.

## 4. Основные этапы разработки

### Этап 1. Проектирование структуры приложения

На начальном этапе была определена архитектура: клиентская часть, серверная часть, база данных и live-взаимодействие через WebSocket. Также были выделены основные сущности: пользователь, квиз, вопрос, попытка, live-сессия.

### Этап 2. Реализация backend и работы с БД

Были реализованы серверные маршруты для регистрации и авторизации, работы с квизами, сохранения попыток, профиля пользователя и live-сессий. На этом этапе также была настроена автоматическая инициализация таблиц в PostgreSQL.

### Этап 3. Разработка интерфейса

Были созданы основные экраны:

- кабинет организатора;
- кабинет участника;
- форма создания и редактирования квиза;
- страница live-квиза для участника;
- страница live-управления для организатора;
- страница профиля.

### Этап 4. Реализация live-режима

Отдельным важным этапом стала реализация live-функциональности:

- создание live-сессии;
- подключение по коду комнаты;
- синхронная выдача вопросов;
- прием ответов участников;
- автоматическое обновление лидерборда;
- завершение сессии с сохранением результатов.

### Этап 5. Повышение надежности и качества

После реализации основной функциональности была проведена доработка качества:

- исправлены критичные серверные дефекты;
- усилены проверки на стороне сервера;
- проект переведен на сборку Vite;
- добавлены unit, integration, page-level smoke и browser e2e тесты;
- снижены риски, связанные с зависимостями и устаревшим tooling-стеком.

## 5. Итог

В результате был разработан рабочий веб-продукт для создания и проведения интерактивных квизов. Приложение поддерживает роли организатора и участника, live-взаимодействие в реальном времени, хранение истории попыток и удобное управление содержимым квизов.

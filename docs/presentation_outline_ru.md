# Структура презентации

## Слайд 1. Название проекта

**Веб-приложение для проведения интерактивных опросов и квизов**

- цель проекта;
- кратко: organizer, participant, live-режим;
- ФИО автора, группа, дисциплина.

## Слайд 2. Проблема и задача

- необходимость удобного инструмента для проведения квизов и опросов;
- раздельные сценарии для организатора и участника;
- потребность в live-формате с подключением по коду комнаты.

## Слайд 3. Что реализовано

- регистрация и авторизация;
- создание и редактирование квизов;
- настройка таймеров, правил и лимитов;
- история попыток;
- профиль и аватар;
- live-сессии с лидербордом.

## Слайд 4. Архитектура решения

- frontend: React + Vite;
- backend: Node.js + Express;
- база данных: PostgreSQL;
- realtime: WebSocket (`ws`).

## Слайд 5. Почему выбран именно такой стек

- React: компонентный подход и удобный интерфейс;
- Vite: быстрая сборка и запуск;
- Express: простой REST API;
- PostgreSQL: надежное хранение данных;
- WebSocket: синхронный live-режим;
- Jest + Playwright: контроль качества.

## Слайд 6. Основные этапы разработки

- проектирование архитектуры и сущностей;
- реализация backend и базы данных;
- разработка экранов organizer и participant;
- реализация live-режима;
- тестирование и доработка надежности.

## Слайд 7. Контроль качества

- unit-тесты;
- integration-тесты backend;
- page-level smoke-тесты;
- browser e2e smoke-тесты.

## Слайд 8. Результат и перспективы

- получен рабочий продукт;
- проект можно расширять новыми типами вопросов, аналитикой и публичным деплоем;
- решение пригодно для учебных, корпоративных и командных сценариев.
